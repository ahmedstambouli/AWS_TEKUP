Lancement d'une instance EC2

[Suivant | Haut]

Effectuez l'une des actions suivantes pour ouvrir la console d'un service (particulirement EC2):

  • Dans la zone de recherche de la barre de navigation, entrez tout ou partie du nom du service.
  • Sous "Console Home", cliquez sur "View all services" et choisissez un nom de service.
  • Dans la barre de navigation, cliquez sur "Services" ensuite, choisissez un service.

Ouvrez la console Amazon EC2 puis, partir du tableau de bord, cliquez sur "Launch Instance".

Nommer l'instance et ajouter des balises (tags)

Les balises permettent la catgorisation de vos instances. Ceci est utile si vous avez plusieurs instances du mme type. Vous pouvez identifier une ressource par sa balise. Chaque balise consiste d'une cl et d'une valeur.

Choisir l'AMI

Une AMI (Amazon Machine Image) est un modle qui contient les configurations logicielles requises pour lancer une instance. Vous pouvez:

  • utiliser les AMI proposes par AWS ou par la communaut
  • crer vos propres AMI personnalises
  • acheter des AMI auprs d'un tiers (AWS marketplace)

Une instance est une copie de l'AMI s'excutant en tant que serveur virtuel dans le Cloud.

partir d'une AMI, on peut lancer une ou plusieurs instances.

Pour l'instance Linux slectionnez l'AMI Amazon Linux 2 AMI (ligible pour loffre gratuite) et pour l'instance Windows slectionnez l'AMI Microsoft Windows Server 2019 Base (ligible pour loffre gratuite).

Choisir le type d'instance

Dans cette tape vous allez slectionner la configuration matrielle de votre instance.

Amazon EC2 fournit plusieurs types d'instances pour diffrents cas d'utilisation. Chaque type d'instance propose diffrentes capacits en terme de:

  • puissance de calcul

Les types d'instance sont regroups en familles d'instance en fonction de ces capacits:

  • usage gnral (A1, T2, M4, etc.)
  • calcul optimis (C4, C5, etc.)
  • mmoire optimise (R6g, X1, mmoire leve, etc.)
  • calcul acclr (P4, G3, etc.)
  • stockage optimis (I3, D2, H1, etc.)

Slectionner la paire de cls

Pour vous connecter votre instance en toute scurit, vous devez utiliser une paire de cls : publique (public key) enregistre chez AWS et prive (private key) enregistre sur votre machine.

Vous tes donc invit choisir une paire de cls existante ou crer une nouvelle paire de cls.

  • Si vous allez crer une nouvelle paire de cl, vous aurez la possibilit de choisir le format de la cl prive (.pem ou .ppk si vous allez utiliser l'outil PuTTY). Le fichier de cl est automatiquement tlcharg et c'est votre seule occasion d'enregistrer le fichier de cl prive.
  • Sinon vous pouvez crer votre paire de cls l'avance en cliquant sur "Key Pairs" dans le volet de navigation gauche. Puis choisissez "Create key pair". Puis lors de la cration de l'instance, slectionnez la paire de cls que vous avez cre.

Configurer les paramtres rseau

Laissez le VPC par dfaut et choisissez le sous-rseau dans lequel votre instance sera cre (sinon un sous-rseau par dfaut dans n'importe quelle zone de disponibilit sera slectionn).

Une instance EC2 ayant une adresse IP publique est accessible via Internet.

Un groupe de scurit joue le rle de pare-feu (firewall) pour les instances associes, en contrlant le trafic entrant et le trafic sortant au niveau de l'instance (en fonction des protocoles et des numros de port).

Les groupes de scurit sont avec tat (stateful) et vous pouvez indiquer des rgles d'autorisation, mais pas des rgles d'interdiction.

Les rgles ajouter seront applicables sur chaque machine associe au groupe de scurit. Vous pouvez modifier ces rgles tout moment (effet immdiat!).

Lorsque vous crez une instance, vous devez l'associer un groupe de scurit.

Si vous ne spcifiez pas un autre groupe de scurit lorsque vous lancez l'instance, celle-ci est automatiquement associe au groupe de scurit par dfaut. Sinon vous pouvez crer un groupe de scurit personnalis lors du lancement de l'instance. Vous pouvez aussi crer un groupe de scurit l'avance en cliquant sur "Security Groups" dans le volet de navigation gauche puis sur "Create security group". Ensuite, lors de la cration de l'instance, slectionnez le groupe de scurit que vous avez cr.

Ajoutez des rgles qui vous permettent de vous connecter votre instance avec :

  • le protocole SSH (Secure Shell) pour l'instance "Amazon Linux 2"
  • le protocole RDP (Remote Desktop Protocol) pour l'instance "Microsoft Windows Server 2019 Base"

Pour des raisons de scurit, ne choisissez pas "Anywhere" comme "Source" avec une rgle SSH ou RDP (cette solution est acceptable pour une dure courte dans un environnement de test, mais n'est pas recommande pour un environnement de production).

Bonne pratique: implmentez les rgles les moins permissives pour votre groupe de scurit.

Ajouter le stockage

Vous allez lancer votre instance en utilisant un disque EBS (Elastic Block Store) qui sera votre disque racine.

Toutes les AMI sont rparties en deux catgories : bases sur Amazon EBS ou bases sur le stockage d'instance. Amazon EBS fournit des volumes de stockage de type bloc qui peuvent persister indpendamment de la dure d'excution d'une instance. Contrairement au stockage d'instances qui fournit un stockage temporaire de type bloc. Les donnes stockes dans un stockage d'instance sont supprimes quand vous arrtez, mettez en veille prolonge ou rsiliez votre instance.

Bonne pratique: utilisez des volumes Amazon EBS distincts pour le systme d'exploitation et pour vos donnes. Assurez-vous que le volume avec vos donnes persiste aprs la terminaison de l'instance (dcocher la colonne "Delete on Termination").

Connexion l'instance

[Prcdent | Suivant | Haut]

La connexion l'instance EC2 distante partir de votre machine locale dpend de votre systme d'exploitation ainsi que l'AMI que vous avez slectionne.

Linux

A partir du tableau de bord EC2 slectionnez votre instance puis cliquez sur "Connect".

Sous Linux et OSX, suivez les instructions listes dans l'onglet "SSH client".

Sous Windows, on va utiliser le client SSH Putty.

Utilisez l'outil PuTTYgen pour convertir votre cl prive (par dfaut .pem) en fichier .ppk afin d'tre accept par Putty. Pour ce faire :

  1. dmarrez PuTTYgen et cliquez sur "Load"
  2. cherchez votre fichier .pem puis cliquez sur "OK"
  3. cliquez sur "Save private key" puis "yes"
  4. donnez un nom votre cl puis cliquez sur "save"

Ouvrez maintenant l'application Putty et procdez comme suit :

  1. saisissez l'adresse IP de votre instance dans la case "Host Name (or IP address)"
  2. sous le menu "SSH" dans le volet de navigation gauche, slectionnez "auth" cliquez sur "Browse" pour choisissez votre cl priv.
  3. cliquez sur "open" puis "yes"

Un terminal PuTTY s'ouvre et vous demande de taper votre nom d'utilisateur (celui de l'instance).

Comme premire interaction avec votre instance, tapez la commande :

In [ ]:
$ sudo yum update

Windows

Vous pouvez vous connecter votre instance Windows l'aide d'un client RDP :

  • Windows : install par dfaut
  • Linux: utilisez Remmina
  • MacOs: utilisez Microsoft Remote Desktop

Ouvrez votre client RDP (Remote Desktop Connexion dans notre cas) et suivez les tapes suivantes pour vous connecter votre instance Windows :

1. Slectionnez votre instance dans la liste des instances puis cliquez sur "Connect"

2. Sur la page "Connect to instance", slectionnez l'onglet "RDP client" puis cliquez sur "Get password" pour obtenir un mot de passe. Il faut attendre quelques minutes aprs le lancement de l'instance pour que ce mot de passe soit disponible.

3. Cherchez votre fichier de cl prive (.pem) que vous avez cr lors du lancement de l'instance et cliquez sur "Open". Puis choisissez "Decrypt Password" pour affichez votre mot de passe.

4. Dans le client RDP vous allez spcifier les valeurs suivantes :

  • Computer : votre "Public DNS"
  • User Name : "Administrator" (ou "Administrateur" pour le franais)
  • Password : le mot de passe que vous avez obtenu

5. Cliquez sur "OK" puis "Yes" pour ouvrir votre instance Windows.

Adressage IP des instances Amazon EC2

[Prcdent | Haut]

Par dfaut, Amazon EC2 utilise le protocole adressage IPv4 (prend en charge aussi le protocole IPv6).

Adresse IP prive Adresse IP publique Adresse IP lastique
N'est pas accessible depuis Internet et sert pour la communication entre les instances de votre VPC (Virtual Private Cloud). Elle reste associe l'interface rseau lorsque l'instance est arrte ou mise en veille puis redmarre. Attribue automatiquement et accessible depuis Internet. Elle est libre lorsque vous associez une adresse IP lastique votre instance ou lorsque l'instance est arrte, mise en veille ou termine. Pour les deux premiers cas, l'instance reoit une nouvelle adresse IP publique au dmarrage. C'est une adresse IPv4 publique statique associe votre compte AWS dans une rgion spcifique. Elle est conserve aprs l'arrt et le dmarrage de votre instance. Son utilisation n'est facture que lorsque elle n'est pas associe une instance en cours d'excution.

Si vous avez besoin d'une adresse IP publique permanente vous devez associez une adresse IP lastique votre instance.

Pour allouer une adresse IP lastique et l'associer votre instance, suivez les tapes suivantes :

  1. Choisissez "Elastic IPs" dans le volet de navigation gauche, puis cliquez sur "Allocate Elastic IP address". Garder la configuration par dfaut et cliquez sur "Allocate"
  2. Cliquez sur "Actions" puis sur "Associate Elastic IP address", slectionnez "Instance" comme "Resource type" et cherchez votre instance dans la liste droulante puis cliquez sur "Associate".
  3. Ouvrez la liste des instances, et vrifiez que votre instance possde maintenant une adresse IP lastique.
