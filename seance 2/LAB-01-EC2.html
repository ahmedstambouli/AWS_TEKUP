LAB-01 : Amazon Elastic Compute Cloud (EC2)¶

L'utilisation d'Amazon Elastic Compute Cloud (Amazon EC2) vous dispense d'investir à l'avance dans du matériel et, par conséquent, vous pouvez développer et déployer les applications plus rapidement. Vous pouvez utiliser Amazon EC2 pour lancer autant d'instances (serveurs virtuels) que nécessaire, configurer la sécurité et les réseaux, et gérer le stockage.

Lancement d'une instance EC2¶

[Suivant | Haut]

Effectuez l'une des actions suivantes pour ouvrir la console d'un service (particulièrement EC2):

  • Dans la zone de recherche de la barre de navigation, entrez tout ou partie du nom du service.
  • Sous "Console Home", cliquez sur "View all services" et choisissez un nom de service.
  • Dans la barre de navigation, cliquez sur "Services" ensuite, choisissez un service.

Ouvrez la console Amazon EC2 puis, à partir du tableau de bord, cliquez sur "Launch Instance".

Nommer l'instance et ajouter des balises (tags)¶

Les balises permettent la catégorisation de vos instances. Ceci est utile si vous avez plusieurs instances du même type. Vous pouvez identifier une ressource par sa balise. Chaque balise consiste d'une clé et d'une valeur.

Choisir l'AMI¶

Une AMI (Amazon Machine Image) est un modèle qui contient les configurations logicielles requises pour lancer une instance. Vous pouvez:

  • utiliser les AMI proposées par AWS ou par la communauté
  • créer vos propres AMI personnalisées
  • acheter des AMI auprès d'un tiers (AWS marketplace)

Une instance est une copie de l'AMI s'exécutant en tant que serveur virtuel dans le Cloud.

À partir d'une AMI, on peut lancer une ou plusieurs instances.

Pour l'instance Linux sélectionnez l'AMI Amazon Linux 2 AMI (éligible pour l’offre gratuite) et pour l'instance Windows sélectionnez l'AMI Microsoft Windows Server 2019 Base (éligible pour l’offre gratuite).

Choisir le type d'instance¶

Dans cette étape vous allez sélectionner la configuration matérielle de votre instance.

Amazon EC2 fournit plusieurs types d'instances pour différents cas d'utilisation. Chaque type d'instance propose différentes capacités en terme de:

  • puissance de calcul
  • quantité de mémoire
  • stockage
  • performances réseau

Les types d'instance sont regroupés en familles d'instance en fonction de ces capacités:

  • usage général (A1, T2, M4, etc.)
  • calcul optimisé (C4, C5, etc.)
  • mémoire optimisée (R6g, X1, mémoire élevée, etc.)
  • calcul accéléré (P4, G3, etc.)
  • stockage optimisé (I3, D2, H1, etc.)

Visitez ce lien pour plus de détails sur les types d'instances EC2.

Choisissez le type d’instance t2.micro (éligible pour l’offre gratuite) qui est sélectionné par défaut.

Sélectionner la paire de clés¶

Pour vous connecter à votre instance en toute sécurité, vous devez utiliser une paire de clés : publique (public key) enregistrée chez AWS et privée (private key) enregistrée sur votre machine.

Vous êtes donc invité à choisir une paire de clés existante ou créer une nouvelle paire de clés.

  • Si vous allez créer une nouvelle paire de clé, vous aurez la possibilité de choisir le format de la clé privée (.pem ou .ppk si vous allez utiliser l'outil PuTTY). Le fichier de clé est automatiquement téléchargé et c'est votre seule occasion d'enregistrer le fichier de clé privée.
  • Sinon vous pouvez créer votre paire de clés à l'avance en cliquant sur "Key Pairs" dans le volet de navigation à gauche. Puis choisissez "Create key pair". Puis lors de la création de l'instance, sélectionnez la paire de clés que vous avez créée.

Configurer les paramètres réseau¶

Laissez le VPC par défaut et choisissez le sous-réseau dans lequel votre instance sera créée (sinon un sous-réseau par défaut dans n'importe quelle zone de disponibilité sera sélectionné).

Une instance EC2 ayant une adresse IP publique est accessible via Internet.

Un groupe de sécurité joue le rôle de pare-feu (firewall) pour les instances associées, en contrôlant le trafic entrant et le trafic sortant au niveau de l'instance (en fonction des protocoles et des numéros de port).

Les groupes de sécurité sont avec état (stateful) et vous pouvez indiquer des règles d'autorisation, mais pas des règles d'interdiction.

Les régles à ajouter seront applicables sur chaque machine associée au groupe de sécurité. Vous pouvez modifier ces régles à tout moment (effet immédiat!).

Lorsque vous créez une instance, vous devez l'associer à un groupe de sécurité.

Si vous ne spécifiez pas un autre groupe de sécurité lorsque vous lancez l'instance, celle-ci est automatiquement associée au groupe de sécurité par défaut. Sinon vous pouvez créer un groupe de sécurité personnalisé lors du lancement de l'instance. Vous pouvez aussi créer un groupe de sécurité à l'avance en cliquant sur "Security Groups" dans le volet de navigation à gauche puis sur "Create security group". Ensuite, lors de la création de l'instance, sélectionnez le groupe de sécurité que vous avez créé.

Ajoutez des règles qui vous permettent de vous connecter à votre instance avec :

  • le protocole SSH (Secure Shell) pour l'instance "Amazon Linux 2"
  • le protocole RDP (Remote Desktop Protocol) pour l'instance "Microsoft Windows Server 2019 Base"

Pour des raisons de sécurité, ne choisissez pas "Anywhere" comme "Source" avec une règle SSH ou RDP (cette solution est acceptable pour une durée courte dans un environnement de test, mais n'est pas recommandée pour un environnement de production).

Bonne pratique: implémentez les règles les moins permissives pour votre groupe de sécurité.

Ajouter le stockage¶

Vous allez lancer votre instance en utilisant un disque EBS (Elastic Block Store) qui sera votre disque racine.

Toutes les AMI sont réparties en deux catégories : basées sur Amazon EBS ou basées sur le stockage d'instance. Amazon EBS fournit des volumes de stockage de type bloc qui peuvent persister indépendamment de la durée d'exécution d'une instance. Contrairement au stockage d'instances qui fournit un stockage temporaire de type bloc. Les données stockées dans un stockage d'instance sont supprimées quand vous arrêtez, mettez en veille prolongée ou résiliez votre instance.

Bonne pratique: utilisez des volumes Amazon EBS distincts pour le système d'exploitation et pour vos données. Assurez-vous que le volume avec vos données persiste après la terminaison de l'instance (décocher la colonne "Delete on Termination").

Détails avancés¶

Pour le moment on ne va pas modifier la configuration avancée de l'instance.

Vérification¶

Sous la section "Summary" vous pouvez examiner et éventuellement modifier les détails de votre instance. Par la suite, cliquez sur le bouton "Launch".

Une page de confirmation indiquant que l'instance est en cours de lancement s'affiche. Sélectionnez "View Instances" pour fermer cette page et afficher la liste des instances EC2.

Connexion à l'instance¶

[Précédent | Suivant | Haut]

La connexion à l'instance EC2 distante à partir de votre machine locale dépend de votre système d'exploitation ainsi que l'AMI que vous avez sélectionnée.

Linux¶

A partir du tableau de bord EC2 sélectionnez votre instance puis cliquez sur "Connect".

Sous Linux et OSX, suivez les instructions listées dans l'onglet "SSH client".

Sous Windows, on va utiliser le client SSH Putty.

Utilisez l'outil PuTTYgen pour convertir votre clé privée (par défaut .pem) en fichier .ppk afin d'être accepté par Putty. Pour ce faire :

  1. démarrez PuTTYgen et cliquez sur "Load"
  2. cherchez votre fichier .pem puis cliquez sur "OK"
  3. cliquez sur "Save private key" puis "yes"
  4. donnez un nom à votre clé puis cliquez sur "save"

Ouvrez maintenant l'application Putty et procédez comme suit :

  1. saisissez l'adresse IP de votre instance dans la case "Host Name (or IP address)"
  2. sous le menu "SSH" dans le volet de navigation à gauche, sélectionnez "auth" cliquez sur "Browse" pour choisissez votre clé privé.
  3. cliquez sur "open" puis "yes"

Un terminal PuTTY s'ouvre et vous demande de taper votre nom d'utilisateur (celui de l'instance).

Comme première interaction avec votre instance, tapez la commande :

In [ ]:
$ sudo yum update

Windows¶

Vous pouvez vous connecter à votre instance Windows à l'aide d'un client RDP :

  • Windows : installé par défaut
  • Linux: utilisez Remmina
  • MacOs: utilisez Microsoft Remote Desktop

Ouvrez votre client RDP (Remote Desktop Connexion dans notre cas) et suivez les étapes suivantes pour vous connecter à votre instance Windows :

1. Sélectionnez votre instance dans la liste des instances puis cliquez sur "Connect"

2. Sur la page "Connect to instance", sélectionnez l'onglet "RDP client" puis cliquez sur "Get password" pour obtenir un mot de passe. Il faut attendre quelques minutes après le lancement de l'instance pour que ce mot de passe soit disponible.

3. Cherchez votre fichier de clé privée (.pem) que vous avez créé lors du lancement de l'instance et cliquez sur "Open". Puis choisissez "Decrypt Password" pour affichez votre mot de passe.

4. Dans le client RDP vous allez spécifier les valeurs suivantes :

  • Computer : votre "Public DNS"
  • User Name : "Administrator" (ou "Administrateur" pour le français)
  • Password : le mot de passe que vous avez obtenu

5. Cliquez sur "OK" puis "Yes" pour ouvrir votre instance Windows.

Test de la connexion à l'aide d'une API¶

[Précédent | Suivant | Haut]

FASTAPI est un Framework moderne et rapide pour la création des API avec Python.

Tester la connexion manuellemnt¶

Dans cette section vous allez installer et utiliser FASTAPI manuellement. Commencez par vérifiez que Python est installé sur votre instance puis installez le Framework FASTAPI et le server Uvicorn:

In [ ]:
$ pip3 install fastapi[all]

$ pip3 install uvicorn[standard]

Créez un fichier "main.py" avec le contenu suivant:

In [ ]:
from fastapi import FastAPI

app = FastAPI()


@app.get("/")
async def root():
    return {"message": "Hello World"}

Lancez le serveur uvicorn avec la commande :

In [ ]:
$ uvicorn main:app --host 0.0.0.0 --port 8000

Sélectionnez votre instance dans le Dashboard EC2 puis cliquez sur le menu "Details" et copiez l'adresse IPv4 de votre instance.

Ouvrez un deuxième onglet de votre navigateur et collez l'adresse http://adresse_IP_de_votre_instance:8000 dans la barre d'adresse.

Automatisation avec les données utilisateur d'instance¶

L'objectif de cette section est de lancer les instructions de la section précédente (installation de fastapi, lancement du serveur uvicorn et exécution du fichier main.py) automatiquement lors de la création de l'instance en utilisant les données utilisateur d'instance (user data). Par défaut, ces données serons exécutées une seule fois au lancement de l'instance.

Vous avez besoin de créer un compte GitLab et d'installer le système de contrôle de version git sur votre machine.

Ouvrez Git Bash (ou votre invite de commande) et tapez la commande suivante pour vérifier l'installation de git :

In [ ]:
$ git --version

Commencez par la configuration de vos paramètres d'accès à GitLab:

In [ ]:
$ git config --global user.name "your_username"
$ git config --global user.email "your_email"

Sur votre machine créez le dossier "awsProject" et y accédez depuis votre Git Bash puis tapez la commande suivante pour l'ajouter à git :

In [ ]:
$ git init

Vous allez constater la création d'un dossier caché ".git" dans votre projet "awsProject".

Créez le fichier "main.py" dans votre dossier "awsProject" avec le contenu suivant:

In [ ]:
from fastapi import FastAPI

app = FastAPI()


@app.get("/")
async def root():
    return {"message": "Hello World"}

Ensuite tapez les commandes suivantes dans votre Git Bash :

In [ ]:
$ git add .
$ git commit -m "my first commit"
$ git push -u https://gitlab.com/your_username/awsproject

Vérifiez la création d'un projet "awsproject" contenant le fichier "main.py" dans votre compte GitLab .

Vous pouvez également commencer par la création d'un projet privé dans GitLab puis pousser votre fichier "main.py" sur votre dépôt distant.

Par la suite, vous aurez besoin d'un jeton d'accès à GitLab. Pour le générer, cliquez sur votre avatar en haut à droite puis sur "Edit profile". Dans le menu à gauche sélectionner "Access Tokens". Donnez un nom et éventuellement une date d'expiration à votre jeton et cochez toutes les portées (scopes) puis cliquez sur "Create personal access token".

Sauvegarder votre jeton dans une emplacement sécurisé car vous n'aurez plus la possibilité de l'afficher.

Procédez à la création d'une nouvelle instance EC2 et sous "Advanced details" spécifiez les données utilisateur (User data) suivantes :

In [ ]:
#!/bin/bash
yum update -y
pip3 install fastapi[all]
pip3 install uvicorn[standard]
yum -y install git
sudo git clone https://gitlab_ci_token:your_access_token@gitlab.com/your_username/awsproject.git
cd awsproject
/usr/local/bin/uvicorn main:app --host 0.0.0.0 --port 8000

Ouvrez un onglet de votre navigateur et collez l'adresse http://adresse_IP_de_votre_instance:8000 dans la barre d'adresse pour afficher le message "Hello Word".

Adressage IP des instances Amazon EC2¶

[Précédent | Haut]

Par défaut, Amazon EC2 utilise le protocole adressage IPv4 (prend en charge aussi le protocole IPv6).

Adresse IP privée Adresse IP publique Adresse IP élastique
N'est pas accessible depuis Internet et sert pour la communication entre les instances de votre VPC (Virtual Private Cloud). Elle reste associée à l'interface réseau lorsque l'instance est arrêtée ou mise en veille puis redémarrée. Attribuée automatiquement et accessible depuis Internet. Elle est libérée lorsque vous associez une adresse IP élastique à votre instance ou lorsque l'instance est arrêtée, mise en veille ou terminée. Pour les deux premiers cas, l'instance reçoit une nouvelle adresse IP publique au démarrage. C'est une adresse IPv4 publique statique associée à votre compte AWS dans une région spécifique. Elle est conservée après l'arrêt et le démarrage de votre instance. Son utilisation n'est facturée que lorsque elle n'est pas associée à une instance en cours d'exécution.

Si vous avez besoin d'une adresse IP publique permanente vous devez associez une adresse IP élastique à votre instance.

Pour allouer une adresse IP élastique et l'associer à votre instance, suivez les étapes suivantes :

  1. Choisissez "Elastic IPs" dans le volet de navigation à gauche, puis cliquez sur "Allocate Elastic IP address". Garder la configuration par défaut et cliquez sur "Allocate"
  2. Cliquez sur "Actions" puis sur "Associate Elastic IP address", sélectionnez "Instance" comme "Resource type" et cherchez votre instance dans la liste déroulante puis cliquez sur "Associate".
  3. Ouvrez la liste des instances, et vérifiez que votre instance possède maintenant une adresse IP élastique.
